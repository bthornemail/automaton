This specification defines automaton execution scripts and TypeScript implementations using RFC 2119 keywords. Automatons provide self-referential execution with Church encoding and dimensional progression.

## 1. Introduction

### 1.1 Purpose

This specification defines automaton execution scripts and implementations that provide self-referential execution with Church encoding and dimensional progression.

### 1.2 Scope

This specification covers:
- Automaton execution scripts
- Continuous automaton implementation
- Ollama automaton implementation
- Advanced automaton engine
- Bootstrap automaton process

### 1.3 RFC 2119 Keywords

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in RFC 2119.

### 2.1 Core Terms

- **Automaton**: Self-referential execution system

## 3. Automaton Types

### 3.1 Continuous Automaton

The system MUST provide:
- **Built-in Intelligence**: Built-in action selection
- **JSONL Loading**: Load automaton from JSONL files
- **Action Execution**: Execute automaton actions
- **State Management**: Manage automaton state

### 3.2 Ollama Automaton

The system MUST provide:
- **AI-Powered Selection**: Ollama-based action selection
- **Model Integration**: Integration with Ollama models
- **Intelligent Decisions**: AI-powered decision making

### 3.3 Advanced Automaton

The system MUST provide:
- **Core Engine**: Core automaton engine
- **Church Encoding**: Church encoding support
- **Dimensional Progression**: Dimensional progression support
- **Self-Reference**: Self-reference pattern support

### 3.4 Bootstrap Automaton

The system MUST provide:
- **Self-Instantiation**: Self-instantiation process
- **Transaction-Based**: Transaction-based bootstrap
- **Validation**: SHACL constraint validation
