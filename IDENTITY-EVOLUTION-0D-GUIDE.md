# Identity Evolution (0D) Guide

## Overview

Identity Evolution (0D) is generated when the automaton executes self-modifications at dimension 0. This represents the foundational quantum vacuum topology and identity processes in the Church encoding system.

## How Identity Evolution (0D) Works

- **Pattern**: Generated by `executeSelfModification()` when `currentDimension === 0`
- **Code Pattern**: Includes `self-evolve`, `vacuum-fluctuation`, `identity-mutation`
- **Frequency**: By default, the automaton cycles through dimensions 0-7, spending only ~12.5% of time at dimension 0

## Configuration Options

### Command-Line Arguments

| Argument | Description | Default |
|----------|-------------|---------|
| `--lock-0d` | Lock to dimension 0 | `false` |
| `--0d` or `--identity-evolution` | Focus on dimension 0 | `false` |
| `--0d-prob=<0-1>` | Probability of returning to dimension 0 | `0.5` (or `0.7` if `--0d` is set) |

### Programmatic Configuration

```typescript

const automaton = new MemoryOptimizedAutomaton('./automaton.jsonl', {
```

## Performance Considerations

### Memory Usage

- Identity Evolution (0D) objects are lightweight (text objects)
- Memory optimization still applies (trimming, GC)
- Locking to dimension 0 reduces overhead from dimension cycling

### Execution Speed

- Locking to dimension 0: **Fastest** (no dimension transitions)
- Focus mode: **Slightly slower** (dimension checks on each cycle)
- Default mode: **Slowest** (full dimension cycling)

## Troubleshooting

### Identity Evolution Not Increasing

1. **Check if locked**: Ensure `--lock-0d` is set or `lockDimension: 0` in config
2. **Check dimension**: Verify `currentDimension === 0` in logs
3. **Check probability**: If using focus mode, ensure probability is high enough
4. **Check execution**: Ensure `executeSelfModification()` is being called

### Too Many Identity Evolution Objects

1. **Enable trimming**: Already enabled in memory-optimized version
2. **Reduce maxObjects**: Lower `maxObjects` in config
3. **Increase trim frequency**: Lower `trimInterval` in config
